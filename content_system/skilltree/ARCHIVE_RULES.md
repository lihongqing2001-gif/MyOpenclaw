# 归档规则

适用范围：`skilltree/` 当前项目目录。

## 目标

- 保留当前可运行版本的快照，避免整理时误覆盖。
- 让后续文件命名、版本留存、归档位置一致。
- 优先非破坏性整理，不直接重命名线上使用中的核心文件。

## 当前目录约定

- `index.html`：技能树展示入口。
- `data.json`：技能树节点数据源。
- `ARCHIVE_RULES.md`：本规则文件。

## 归档规则

### 1. 快照归档

每次执行“自动整理归档”时：

- 将当前版本复制到 `../07_archive/skilltree/YYYY-MM-DD-HHMM/`
- 归档内容至少包含：
  - `index.html`
  - `data.json`
  - `manifest.txt`

### 2. 命名规范

- 主运行文件保持稳定命名：`index.html`、`data.json`
- 历史版本不直接堆在工作目录，统一进入 `../07_archive/skilltree/`
- 归档目录使用时间戳命名：`YYYY-MM-DD-HHMM`

### 3. 去重策略

- 若新快照与最近一次归档内容完全一致，后续可跳过重复归档。
- 判断依据优先使用 SHA-256 校验值。
- 未经确认，不删除任何历史归档。

### 4. 目录重构原则

- 当前工作目录仅保留“最新可编辑版本”。
- 历史版本、实验稿、导出稿统一转移到 `../07_archive/skilltree/`。
- 若后续文件增多，可扩展为：
  - `src/`：源数据或脚本
  - `dist/`：导出产物
  - `docs/`：说明文档
  - `../07_archive/skilltree/`：历史快照
