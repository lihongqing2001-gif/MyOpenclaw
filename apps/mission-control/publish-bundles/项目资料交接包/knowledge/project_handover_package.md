# SOP: 项目资料交接包

## Goal
把一个目录、项目或仓库整理成可以直接交给别人继续接手的交接包。

## Default Trigger
- “给这个项目做一份交接包”
- “把这个目录整理成交接文档”
- “让别人一看就知道怎么继续做”

## Inputs
- 目标目录
- 交接对象
- 完整度等级

## Preconditions
- 目录可访问
- `文件与版本管理` 可用
- 允许写入目标目录或 `outputs/`

## Output Contract
默认输出：
- `<目标目录>/HANDOVER.md`
- `<目标目录>/HANDOVER_CHECKLIST.md`

至少包含：
- 项目目标
- 当前状态
- 目录说明
- 已知风险
- 下一步建议

## Steps
1) 盘点目标目录
2) 提取关键文档和结构
3) 说明当前状态与已完成项
4) 列出未完成项与风险
5) 生成交接文档与检查清单

## Failure Handling
- 如果目录太大，先索引再抽样
- 如果缺关键文件，明确标注而不是假设完整
