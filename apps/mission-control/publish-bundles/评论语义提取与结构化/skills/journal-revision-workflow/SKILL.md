# Journal Revision Workflow

Guide the manuscript from comment intake through a QA-ready resubmission package.

## When To Use

Trigger this skill when a journal sends editorial feedback, reviewer comments, or formatting corrections that must be tracked across the manuscript, response letter, and final delivery files.

## Workflow

1. Capture every comment before drafting changes. Record source, comment id, priority, due date, and affected manuscript section in a single intake table.
2. Build the revision matrix from the intake list. Each row should map one comment to one action plan, one owner, one status, and one manuscript location.
3. Split work into three streams:
   - manuscript edits in Word with Track Changes
   - response letter drafting
   - citation or evidence verification when claims are added or revised
4. Keep the status tracker current after each pass so blocked items, coauthor dependencies, and ready-for-review items are visible.
5. Prepare the Word handoff package:
   - clean manuscript
   - tracked manuscript
   - response letter draft
   - short handoff memo listing what changed, what remains open, and which file is authoritative
6. Run a pre-resubmission QA pass. Check figure and table numbering, cross-references, abbreviations, units, citations, supplementary references, and page or section references cited in the response letter.
7. Before delivery, confirm every reviewer comment is answered, every answer points to a real manuscript change or justified rebuttal, and the tracker marks the package as ready.

## Non-Negotiable Rules

- Do not drop or merge reviewer comments unless the mapping remains explicit.
- Do not claim a manuscript change in the response letter unless it exists in the tracked manuscript.
- Do not mark a row complete while the response letter or citation evidence is still pending.
- Escalate comments that could change methods, data interpretation, ethics statements, or authorship.

## Templates

Use these files under `templates/`:

- `revision-matrix.md` for comment-to-action mapping
- `response-letter.md` for editor and reviewer replies
- `status-tracker.md` for progress, blockers, and handoff readiness

If the working folder already has CSV or Markdown trackers, align field names instead of duplicating the same information.
