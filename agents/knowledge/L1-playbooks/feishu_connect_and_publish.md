# 飞书连接并发布应用

这是当前确认过的正确顺序，后续 OpenClaw 处理飞书应用接入与发布时，必须优先按这个流程执行。

## 正确顺序

### 1. 配置事件与回调

- 在飞书开发者后台打开应用配置
- 进入事件与回调
- 订阅方式选择 `长连接接收事件`
- 保存

### 2. 添加事件订阅

- 添加事件
- `通讯录相关` 全部打勾
- `消息与群组`
  - 接收消息
  - 机器人进群
- 确认保存

### 3. 创建并发布版本

- 返回飞书开发者后台
- 创建版本
- 版本号填写 `1.0.0`
- 更新说明可先简写
- 保存
- 确认发布

### 4. 在飞书内验证

- 打开飞书 App
- 等待收到应用审批通过通知
- 打开应用
- 开始和你的“小龙虾”对话

## 强约束

- 事件订阅优先使用长连接，不要默认选 HTTP 回调
- 发布前必须确认通讯录事件与消息/群组事件已经勾选
- 不要跳过 `创建版本 -> 发布` 这一步
- 最终验证必须在飞书 App 内完成，而不是只看开发者后台

## 给 OpenClaw 的执行提示

- 如果用户要求“接飞书”或“发布飞书应用”，先按本条 playbook 执行
- 如果缺权限或后台配置入口打不开，逐项向用户报告卡在哪一步
- 发布完成后，给出一条面向人的最短验证路径：
  - 打开飞书
  - 找到应用审批通知
  - 进入应用
  - 发送一条测试消息
